Theorie

Vektor vs. Pixel — und die Grundregeln für Icons & Bilder

SVG (Vektor) beschreibt Formen mathematisch — skaliert verlustfrei, ideal für Icons und Logos. PNG/JPG/WebP (Pixel) speichern ein Farbpunktraster — ideal für Fotos. Die richtige Format-Wahl spart Dateigröße und verhindert unscharfe Darstellungen.

6 Grundregeln

SVG für alles Gezeichnete

Icons, Logos, Illustrationen → immer SVG. Skaliert auf jede Pixeldichte, Farbe und Stroke per CSS steuerbar.

WebP/JPG für Fotos

Fotos als WebP (beste Kompression) oder JPG. PNG nur bei Transparenz-Bedarf. Niemals Fotos als SVG.

Stroke-Icons: Leicht und modern

Bestehen aus offenen Linien. Ideal neben Text. Stroke-Width muss zur Größe passen — zu dünn = fragil, zu dick = klobig.

Fill-Icons: Schwer und auffällig

Mehr visuelles Gewicht. Ideal für aktive Zustände (z.B. gefüllter Stern) und alleinstehende Icons ohne Text.

object-fit ist dein bester Freund

User-Bilder haben unvorhersehbare Proportionen. Feste Container + object-fit: cover verhindert Layout-Brüche.

Text auf Bildern braucht Overlay-Schutz

Bilder sind dynamisch — weißer Text auf hellem Bereich verschwindet. Gradient-Overlays sichern konsistenten Kontrast.

Analyse

Links: Gleiche Stroke-Width überall — das 16px-Icon wirkt erdrückt, das 32px wirkt dünn. Rechts: Angepasste Strichstärke — optisch gleichwertiges Gewicht trotz unterschiedlicher Größe.

Regel

Stroke-Width zur Icon-Größe passen. 16px→1.5, 20px→1.75, 24px→1.75–2.0, 32px+→2.0. Lieber zu dünn als zu dick.

Empfehlung

Für React: npm install lucide-react — 1400+ konsistente Stroke-Icons, tree-shakeable, konfigurierbar.

⚠ Opinionated

Lucide ist eine von vielen guten Optionen. Alternativen: Heroicons (Tailwind), Phosphor, Tabler, Radix Icons. Das Wichtigste ist Konsistenz innerhalb deines Projekts, nicht welche Library du wählst.

Analyse

Links: Bilder behalten natürliche Höhe — Grid bricht. Rechts: Feste Container-Höhe + overflow hidden. cover füllt komplett (schneidet ab), contain zeigt alles (mit Letterbox), aspect-ratio für responsive Proportionen.

Regel

Immer feste Container (oder aspect-ratio) + object-fit: cover für Bilder-Grids. Niemals Bilder mit natürlicher Größe in Layouts, die Konsistenz brauchen.

Regel

Text auf Bildern braucht immer Kontrasthilfe: Gradient-Overlay, Vollfarben-Overlay oder text-shadow. Bei User-Uploads hast du keine Kontrolle über den Inhalt.
